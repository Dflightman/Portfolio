PHASE 1

📁 Project Setup & File Structure

Step 1.1 — Create Your Project Folder

Open VS Code. Press Ctrl + ` to open the terminal. Type:

$ cd Desktop
$ mkdir portfolio
$ cd portfolio
$ code . # Opens this folder in VS Code

Step 1.2 — Your Final File Structure

Your folder should look like this when everything is done:

📁 portfolio/ (your project root)
  ├── index.html ← the website
  ├── styles.css ← all the styling
  ├── script.js ← password checker + form JS
  ├── .gitignore ← tells Git what to ignore
  ├── README.md ← project description (create this)
  ├── 📁 .github/
    └── 📁 workflows/
      └── deploy.yml ← CI/CD pipeline
  └── 📁 backend/
    ├── server.js ← Node.js + Express server
    └── package.json ← backend dependencies

Step 1.3 — Copy the Provided Files

Copy all the files provided (index.html, styles.css, script.js, .gitignore, backend/server.js, backend/package.json, and .github/workflows/deploy.yml) into the correct locations in your portfolio folder.

💡 Tip: In VS Code, you can drag and drop files into the Explorer panel on the left, or use File → New File to create them.
PHASE 2

🔧 Git Installation & Setup

Step 2.1 — Download & Install Git

  1. Go to https://git-scm.com/download/win
  2. Download the installer for Windows (64-bit)
  3. Run the installer — click Next on every screen (default settings are fine)
  4. Click Finish

Step 2.2 — Verify Git is Installed

Open a new terminal in VS Code and type:

$ git --version
git version 2.x.x ← you should see something like this

Step 2.3 — Configure Your Git Identity

Tell Git who you are. Replace with your own name and email:

$ git config --global user.name "Dan Joseph"
$ git config --global user.email "your.email@gmail.com"
$ git config --global init.defaultBranch main

# Confirm your settings:
$ git config --list

Step 2.4 — Initialize Git in Your Project

Make sure you are inside your portfolio folder, then run:

$ git init
Initialized empty Git repository in .../portfolio/.git/

Step 2.5 — Your First Commit

$ git add . # Stage ALL files
$ git status # See what was staged
$ git commit -m "🚀 Initial commit — portfolio + password checker"
✓ You now have a local Git repository with your first commit saved!
PHASE 3

🐙 Push to GitHub

Step 3.1 — Create a GitHub Account

  1. Go to https://github.com and sign up (it's free!)
  2. Verify your email address
  3. Remember your username — you'll use it everywhere

Step 3.2 — Create a New Repository

  1. Click the + button → New repository
  2. Repository name: portfolio
  3. Description: BCA Portfolio with Password Strength Checker
  4. Set it to Public (required for free GitHub Pages)
  5. DO NOT check "Add a README" or "Add .gitignore" — we already have those
  6. Click Create repository

Step 3.3 — Connect Local Git to GitHub & Push

GitHub will show you commands. Use these (replace YOUR_USERNAME):

# Connect your local repo to GitHub:
$ git remote add origin https://github.com/YOUR_USERNAME/portfolio.git

# Push your code to GitHub:
$ git push -u origin main

# GitHub will ask for your username and password.
# For password: use a Personal Access Token (see step 3.4)

Step 3.4 — Create a GitHub Personal Access Token

GitHub no longer accepts passwords for pushing. Create a token instead:

  1. Go to GitHub → click your profile picture → Settings
  2. Scroll down to Developer settings (bottom of left sidebar)
  3. Click Personal access tokens → Tokens (classic)
  4. Click Generate new token (classic)
  5. Note: VS Code push, Expiration: 90 days
  6. Check the box: repo (gives full repo access)
  7. Click Generate token
  8. ⚠️ COPY THE TOKEN NOW — you won't see it again!
  9. Use this token as your "password" when Git asks
⚠️ Save your token in Notepad temporarily. Never share it with anyone!

Step 3.5 — Verify Your Code is on GitHub

Visit https://github.com/YOUR_USERNAME/portfolio — you should see all your files there!

PHASE 5

⚙️ CI/CD with GitHub Actions

The .github/workflows/deploy.yml file is already set up for you! Here's what it does:

  • Runs automatically every time you git push to main
  • Job 1 (Test): Checks all files exist, validates Node.js syntax
  • Job 2 (Deploy): Only runs if tests pass → deploys to GitHub Pages

Step 5.1 — Trigger the Pipeline

Make any small change (e.g., add a space to README.md) and push:

$ git add .
$ git commit -m "⚙️ trigger CI/CD pipeline"
$ git push

Step 5.2 — Watch Your Pipeline Run

  1. Go to your GitHub repository
  2. Click the Actions tab
  3. You'll see your workflow running — green ✓ means success!
  4. Click on the workflow to see detailed logs
✓ Every git push now automatically tests and deploys your site!

Step 5.3 — The Workflow Explained

on: push → main # Triggered by: git push
jobs:
  test: # Job 1: run checks
    - checkout code
    - install dependencies
    - verify files exist
    - check JS syntax
  deploy: # Job 2: only runs if test passes
    - needs: test
    - deploy to GitHub Pages

PHASE 7

☁️ Backend Hosting on Render.com

FREE TIER No credit card needed for the basics!

Step 7.1 — Create a Render Account

  1. Go to https://render.com
  2. Click Get Started for Free
  3. Sign up with your GitHub account (easiest option)

Step 7.2 — Create a PostgreSQL Database on Render

  1. On Render dashboard, click New +
  2. Select PostgreSQL
  3. Name: portfolio-db
  4. Region: Singapore (closest to India)
  5. Plan: Free
  6. Click Create Database
  7. Wait 1-2 minutes for it to provision
  8. Scroll down to Connections section
  9. Copy the Internal Database URL — it looks like:
    postgresql://user:password@host/dbname

Step 7.3 — Deploy the Node.js Backend

  1. On Render dashboard, click New + → Web Service
  2. Connect your GitHub repository: select portfolio
  3. Configure:
    • Name: dan-portfolio-api
    • Region: Singapore
    • Branch: main
    • Root Directory: backend
    • Build Command: npm install
    • Start Command: node server.js
    • Plan: Free
  4. Click Advanced → Add Environment Variable:
    • Key: DATABASE_URL
    • Value: paste the Internal Database URL from step 7.2
  5. Also add:
    • Key: NODE_ENV
    • Value: production
  6. Click Create Web Service
  7. Wait 3-5 minutes for deployment

Step 7.4 — Get Your Backend URL

After deployment, Render gives you a URL like:

https://dan-portfolio-api.onrender.com

Visit this URL in your browser — you should see the JSON status response!

⚠️ Free tier note: Render's free tier spins down after 15 minutes of inactivity. The first request after that takes 30-60 seconds to wake up. This is normal!

🆘 Troubleshooting Common Issues

ISSUE 01
❌ "git is not recognized as a command"

Git wasn't installed correctly. Re-download from git-scm.com, run installer, then restart VS Code completely.

ISSUE 02
❌ GitHub push asks for username/password repeatedly

Use a Personal Access Token as your password (see Step 3.4). Or run: git config --global credential.helper store to save credentials.

ISSUE 03
❌ GitHub Actions failing (red X)

Click on the failed job → read the error message. Most common fix: make sure all file names match exactly (index.html not Index.HTML).

ISSUE 04
❌ Contact form shows "Backend not connected"

Check: (1) BACKEND_URL in script.js is your correct Render URL, (2) Your Render service is running (not sleeping — send it a request first), (3) CORS origin in server.js matches your GitHub Pages URL.

ISSUE 05
❌ Render backend "Database connection error"

Go to Render dashboard → your web service → Environment → check that DATABASE_URL is set correctly. Copy the Internal URL from your PostgreSQL service on Render.

ISSUE 06
❌ GitHub Pages shows 404

Go to repository Settings → Pages → make sure Source is set to "GitHub Actions". Wait 3 minutes, then try again.
